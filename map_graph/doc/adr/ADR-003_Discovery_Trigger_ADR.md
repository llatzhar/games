# ADR-003: 新都市発見トリガーの設計

## Status

**Proposed** - 2026-01-28

---

## Context

ローグライク要素として「ターンが経過すると新たな敵の国が発見されて増えていく」という要件がある。この「発見」がいつ・どのような条件で発生するかは、ゲームのテンポと難易度に直接影響する重要な設計決定である。

### 考慮すべき要素

1. **テンポ**: 発見が頻繁すぎると対処が追いつかない、稀すぎると退屈
2. **緊張感**: 予測不能性がローグライクらしさを生む
3. **難易度スケーリング**: ゲーム後半は強力な敵が登場すべき
4. **プレイヤー進行度との連動**: うまくいっているプレイヤーにはより多くの挑戦を

---

## Decision

**「固定間隔 + 確率 + 進行度補正」のハイブリッド方式を採用する**

### トリガーロジック

```
毎ターン終了時:
  if 現在ターン >= 次の発見チェックターン:
    確率 = BASE_PROBABILITY + (占領都市数 × PROGRESS_BONUS)
    if random() < 確率:
      新都市を発見
      次の発見チェックターン = 現在ターン + BASE_INTERVAL ± VARIANCE
    else:
      次の発見チェックターン += 1  # 次ターンに再チェック
```

### パラメータ

| パラメータ | 初期値 | 説明 |
|-----------|-------|------|
| `BASE_INTERVAL` | 4ターン | 基本発見間隔 |
| `INTERVAL_VARIANCE` | ±2ターン | 間隔のランダム揺らぎ |
| `BASE_PROBABILITY` | 0.5 | 基本発見確率 |
| `PROGRESS_BONUS` | 0.05/都市 | 占領都市数による確率上昇 |
| `POWER_SCALING` | 1.0 + 0.1 × ターン | 新敵国の強さ係数 |

---

## Considered Options

### Option A: 固定ターン間隔

```
if ターン数 % FIXED_INTERVAL == 0:
    新都市を発見
```

**長所:**
- 完全に予測可能でプレイヤーが計画しやすい
- 実装が最もシンプル
- バランス調整がしやすい

**短所:**
- 予測可能すぎてローグライク感がない
- 毎回同じリズムで単調になりやすい

**却下理由:** ローグライクの「予測不能な展開」という体験が損なわれる。

---

### Option B: 完全確率ベース

```
毎ターン終了時:
    if random() < DISCOVERY_PROBABILITY:
        新都市を発見
```

**長所:**
- 完全に予測不能
- シンプルな実装

**短所:**
- 連続発生で難易度急上昇の可能性
- 長期間発生しない可能性（プレイヤーが暇になる）
- 確率分布が偏ると極端な状況が発生

**却下理由:** 体験のばらつきが大きすぎ、安定したゲーム体験を提供できない。

---

### Option C: プレイヤーアクション連動

```
都市を占領したとき:
    確率判定で新都市発見
```

**長所:**
- プレイヤーの行動に直接フィードバック
- 積極的なプレイが報われる/罰せられる

**短所:**
- 消極的なプレイで発見を回避できてしまう
- ゲームテンポがプレイスタイルに依存しすぎる
- 占領ペースとマップ拡張ペースが連動し、バランス調整困難

**却下理由:** 消極的プレイが最適戦略になりうる問題。

---

### Option D（採用）: 固定間隔 + 確率 + 進行度補正

**長所:**
- 最低保証間隔で急激な難易度上昇を防止
- 確率要素でローグライク感を維持
- 進行度連動で上手いプレイヤーにチャレンジを提供
- 複数のパラメータで細かな調整が可能

**短所:**
- 実装がやや複雑
- パラメータ調整に手間がかかる
- ロジックの説明がプレイヤーに伝わりにくい

---

## Consequences

### 体験面

| 影響 | 詳細 |
|-----|------|
| ✅ 緊張感 | 「いつ来るかわからない」が最低間隔で緩和された不安感 |
| ✅ テンポ | 間隔保証で対処時間が確保される |
| ✅ 適応的難易度 | 上手いプレイヤーほど多くの敵が出現 |
| ⚠️ 透明性 | ロジックがプレイヤーに見えにくい |

### 技術面

| 影響 | 詳細 |
|-----|------|
| ✅ 調整容易性 | 4つの独立したパラメータで細かく調整可能 |
| ✅ テスト容易性 | シード固定で再現可能 |
| ⚠️ バランス工数 | パラメータの相互作用を考慮した調整が必要 |
| ✅ 拡張性 | 追加のトリガー条件（イベント等）を組み込みやすい |

### 調整のガイドライン

| 目的 | 調整方法 |
|-----|---------|
| 難易度を下げる | `BASE_INTERVAL`↑, `BASE_PROBABILITY`↓ |
| ローグライク感を強める | `INTERVAL_VARIANCE`↑, `PROGRESS_BONUS`↑ |
| 序盤を楽にする | 初期数ターンは発見禁止期間を設ける |
| 終盤を厳しくする | `POWER_SCALING`↑ |

---

## 将来の拡張

- **イベントシステム連動**: 特定イベントで強制発見
- **難易度設定連動**: Easy/Normal/Hardでパラメータセット切り替え
- **ナラティブ統合**: 発見時に敵国のフレーバーテキスト表示
- **前兆システム**: 発見の1〜2ターン前に「兆候」を表示
