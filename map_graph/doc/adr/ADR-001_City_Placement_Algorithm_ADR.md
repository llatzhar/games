# ADR-001: 新都市配置アルゴリズムの選択

## Status

**Proposed** - 2026-01-28

---

## Context

ローグライク要素として、ゲーム進行に伴い新しい敵国の都市がマップに追加される。この「発見」を自然かつ戦略的に意味のある形で実現するため、新都市の配置アルゴリズムを決定する必要がある。

### 制約条件

1. 新都市は既存のグラフ構造の「外側」に配置される
2. 既存都市と道路で接続できる距離に配置する
3. 既存都市と近すぎず、遠すぎない位置
4. マップ全体が一方向に偏らないことが望ましい

### プレイヤー体験への影響

- 「未知の領域を発見する」という探索感
- 戦略的に意味のある位置（防衛ラインの外側等）

---

## Decision

**「重心からの放射配置 + ランダム揺らぎ」方式を採用する**

### アルゴリズム概要

1. 既存都市群の重心を計算
2. ランダムに選んだ基準都市から、重心と逆方向（外側）に向かうベクトルを算出
3. ベクトルに角度揺らぎを加え、距離範囲内で候補位置を決定
4. 既存都市との最小距離制約を検証
5. 失敗時はリトライ、最終的にフォールバック処理

---

## Considered Options

### Option A: 完全ランダム配置

```
新都市位置 = ランダム座標（画面範囲内）
```

**長所:**
- 実装が最もシンプル
- 予測不能性が高い

**短所:**
- 既存グラフと離れすぎる/近すぎる可能性
- マップが散らばって視認性悪化
- 「発見」の自然さが損なわれる

**却下理由:** 戦略性と視覚的一貫性が著しく低下するため。

---

### Option B: グラフ境界上配置（Convex Hull）

```
新都市位置 = 既存都市のConvex Hull外周上のランダム点
```

**長所:**
- 必ず「外側」になることが数学的に保証
- 形状がきれいに広がりやすい

**短所:**
- Convex Hull計算のオーバーヘッド
- 都市が必ず境界上に並び、不自然な直線配置になりやすい
- 内側への配置ができない（孤立した内部都市が作れない）

**却下理由:** 形状が機械的になりすぎ、有機的なマップ形状が作れない。

---

### Option C: 既存道路の延長配置

```
新都市位置 = 既存道路の端点から道路方向に延長した位置
```

**長所:**
- 道路ネットワークの自然な延長
- 接続の自明性が高い

**短所:**
- 道路の端点という概念がグラフ上では曖昧
- 既存道路密度が高い領域に偏る可能性
- 分岐パターンが単調になりやすい

**却下理由:** 配置の多様性が限定されすぎる。

---

### Option D（採用）: 重心からの放射配置 + ランダム揺らぎ

```
重心 = 全都市座標の平均
方向 = (基準都市 - 重心).normalize() + ランダム角度揺らぎ
新都市 = 基準都市 + 方向 * ランダム距離
```

**長所:**
- 自然に外側へ広がる傾向
- 揺らぎで有機的な形状
- パラメータで制御しやすい
- 実装難易度が適切

**短所:**
- 重心計算が毎回必要（ただし軽量）
- 都市数が少ないと方向が偏る可能性

---

## Consequences

### 体験面

| 影響 | 詳細 |
|-----|------|
| ✅ 探索感 | 外側に広がるため「未知の領域」感が自然に表現される |
| ✅ 戦略性 | 既存領土の外縁に敵が現れ、防衛ラインの意識が生まれる |
| ✅ 視認性 | パラメータで距離制御できるため、マップが見やすく保てる |
| ⚠️ 予測可能性 | 慣れたプレイヤーは大まかな出現方向を予測できる（意図的かもしれない） |

### 技術面

| 影響 | 詳細 |
|-----|------|
| ✅ 実装容易性 | ベクトル演算のみで実装可能、依存ライブラリ不要 |
| ✅ 調整容易性 | 角度揺らぎ・距離範囲がパラメータ化されている |
| ⚠️ エッジケース | 都市が3つ未満の場合の重心計算に注意が必要 |
| ⚠️ リトライ上限 | 候補が見つからない場合のフォールバック実装が必要 |

### 将来の拡張

- 地形概念追加時、地形を考慮した配置に拡張可能
- 「戦略的重要地点」への意図的配置ロジックを追加可能
