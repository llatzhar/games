Roguelike Strategy Game
=======================

# 概要

* ボードゲームのPandemicのようなターン制システム
  * 自軍(プレイヤ)がn回行動すると、敵軍(CPU)がm回行動する
  * 自軍・敵軍ともに行動回数分、軍ユニットを都市から都市へ移動できる
  * 自軍・敵軍ともに行動回数1回を消費して、都市上に軍ユニットを編成できる
* 軍ユニット
  * 軍ユニットは兵数をもつ。兵数が0になると軍ユニットは消失する
  * 空いたリーダーがいる場合、軍ユニットの編成時にリーダーを割り当てることができる
  * 軍ユニットが兵数0で消失するとリーダーは無所属となり、次の軍ユニット編成時に割り当てに使用できる
* ボードゲームのPandemicのようなマップ
  * 都市と都市が道路でつながるグラフ
* ローグライクにマップを拡張
  * 自国の都市は2つ敵国都市1つから始まり、ターンが経過すると新たな敵の国が発見されて増えていく
* 敵との交戦
  * 都市は軍ユニットを供出する
  * 軍ユニットは都市から道路のつながった都市へ移動できる
  * 都市には軍ユニットが滞在できる
  * 都市に自軍と敵軍双方が滞在すると戦闘が行われる
  * 都市に他の国の軍が滞在すると占領が始まる
  * 占領は軍ユニットの兵数依存で進み、1ターンで終わらないときは次ターンに持ち越す
  * 占領が完了すると都市が所属する国が変わる
  * 自国所属の都市は自軍ユニットを1つ提供する
  * 敵国の都市をすべて占領するのを目的とする
* リーダー
  * 自軍が他国の都市を解放するとリーダーを一人獲得できる
  * リーダーは個別にステータス値を持つ
  * 軍ユニットはリーダーに率いられる。軍の性能にはリーダーのステータス値が乗算される
* (ゲーム性) 自軍が生存して勢力を拡大すると強力になるが、ターンが経過すると強力な敵勢力が登場する


# 設計方針

* pythonを使用、pyxelによる表現
* キーボード＆マウス操作によるユーザ入力
* オフラインに閉じたシステム
* 画面遷移に階層化StateMachineを採用。モーダルダイアログなどのためにPushdownの仕組みも
* モデル・ゲームオブジェクトのjsonファイルシリアライズによるデータセーブ

## コアゲームループ

1. プレイヤーターン
   1. コマンド実行(ユニット移動|ユニット雇用)
   2. 戦闘処理(兵力0の軍ユニットは消失)
   3. コマンド数が残っていれば1-1.へ
2. マップ処理(占領実行、新都市発見、)
3. CPUターン
   1. コマンド実行
   2. 戦闘処理
   3. コマンド数が残っていれば2-1.へ

## マップ生成のアルゴリズム

* 初期状態: 自軍2都市、敵軍1都市を正三角形に配置
* 新しい都市は既知の都市と道の外側に生成される
* 新しい都市から既知の都市に1－3本の道を生成する。新しい道は既存の道と交差してはならない
* 道の長さは一定以内に制限してマップ全体が把握しづらくならないように
